# ChemAI: Predict the Cure

Решение хакатона по предсказанию противовирусной активности молекул.
Для каждого соединения предсказываем три значения: `IC50, mM`, `CC50, mM` и `SI`.
Метрика — средний RMSE по трём таргетам на исходной шкале.

Финальный submission: `submissions/submission_raw_best_cc50ext_b15.csv`,
public score около **264.30**(в разные запуски был также **264.35**).

## Подход

**Данные.** Train — 751×210 RDKit-дескрипторов, test — 250×210. 18 константных
колонок выбрасываем, остаётся 192 признака. Пропусков мало (меньше 30 ячеек суммарно),
импьютим медианой внутри CV-fold через
`SimpleImputer(strategy="median", add_indicator=True)` в составе `Pipeline` —
утечки медиан между фолдами нет.

**Валидация.** В train есть повторяющиеся векторы дескрипторов: 60 групп с
дубликатами, до 16 строк в одной группе. Поэтому используем
`GroupKFold(n_splits=5)` по хешу дескрипторов — одинаковые молекулы не попадают
одновременно в train- и valid-fold. Это даёт строгий, но честный OOF.

**Модели.** Три параллельные ветки, по одной на target-transform:


| ветка | модели                              | используем для |
| ----- | ----------------------------------- | -------------- |
| raw   | KernelRidge + ExtraTrees            | IC50, CC50     |
| sqrt  | CatBoost + ExtraTrees + KernelRidge | SI             |
| log1p | CatBoost + SVR                      | SI             |


Все три таргета с длинным правым хвостом (p99 в десятки раз больше медианы),
поэтому в SI смешиваем sqrt- и log1p-ветки: `SI = sqrt + 1.02 * (log1p − sqrt)`,
клип в [0, 500].

**Постобработка.** Для IC50 cap на 2200: в test p99 = max = 2200, верхний клип
совпадает с верхним наблюдаемым уровнем и срезает только редкие выбросы предсказаний.
Никаких индивидуальных правок строк (например, по `index=141`) или exact-match по
дубликатам в финал не входит — пробовали, выигрыш маленький.

**ChEMBL как мета-признак (только для CC50).** Внешний датасет не используется
как ответы для test. Схема такая:

- из ChEMBL берём ~2000 CC50-like измерений (типы CC50/TC50/LC50, strict equality),
приводим единицы к µM, считаем RDKit-дескрипторы по тем же колонкам, что и в
соревновании, и агрегируем медианой по canonical SMILES;
- обучаем `ExtraTrees(log1p)` на этих данных и получаем `external_cc50_prediction`
для train/test;
- добавляем этот столбец как дополнительный признак, переобучаем
`ExtraTrees(log1p)` уже на competition train и предсказываем CC50;
- финальный CC50 = `(1 − β) * base + β * ext`, β = 0.15.

Прямое добавление ChEMBL-строк в train ухудшало OOF CC50 (557.8 vs 500.7).
Схема через мета-признак нейтральна на OOF (500.1) и даёт небольшой стабильный
выигрыш на public.

## Результаты


| submission                                     | OOF GroupKFold (raw) | public |
| ---------------------------------------------- | -------------------- | ------ |
| baseline (raw IC50/CC50 + SI blend + cap 2200) | 510.9                | 265.31 |
| + ChEMBL CC50 meta, β = 0.15 (финал)           | —                    | 264.30 |


OOF заметно строже public: GroupKFold по дубликатам отделяет похожие молекулы,
public-test — стандартный random split на маленькой выборке (751 строка). Разрыв
ожидаемый. При выборе финальной версии смотрели одновременно на OOF и public:
вариант с β = 0.22 даёт public немного лучше (264.19), но менее стабилен между
запусками, поэтому основной — β = 0.15.

## Внешние данные

Использование внешних данных согласовано с преподавателем: разрешено при условии,
что в репозитории явно описано их происхождение. Ниже — источник, фильтры
и пайплайн подготовки.

**Источник.** Публичная база ChEMBL (release 34), выгрузка через
официальный `chembl_webresource_client`.

**Что брали.** Активности типов `CC50`, `TC50`, `LC50` со `standard_relation = "="`
из assay'ев, описание которых содержит маркеры цитотоксичности (CC50, cytotoxicity,
cell viability, MTT, MDCK/Vero cytotoxicity) и не содержит маркеров отношения
типа «CC50/EC50», «therapeutic index», «selectivity index» — это исключает
производные индексы и оставляет только прямые CC50-измерения.

**Что делали.** Привели единицы к µM (поддерживаются `pM`/`nM`/`µM`/`mM`),
по `canonical_smiles` пересчитали RDKit-дескрипторы ровно по тому же списку
колонок, что и в соревновании, и агрегировали медианой по молекуле. В итоге —
~2000 уникальных CC50 на молекулярных дескрипторах в той же схеме, что и train.

**Как использовали.** Не как дополнительные строки train (это пробовали — OOF CC50
ухудшалось с 500.7 до 557.8). Вместо этого — как мета-признак: отдельная
`ExtraTrees`-модель учится на ChEMBL и даёт `external_cc50_prediction`,
который добавляется единственной дополнительной колонкой к признакам competition
train. Соревновательные target-значения test не используются нигде.

Готовый CSV лежит в `external_data/`. Код полной пересборки находится в ноутбуке
(раздел 10), по умолчанию выключен флагом `RUN_CHEMBL_PREPROCESSING = False`,
чтобы основной запуск не требовал сети и `rdkit`.

## Воспроизводимость

- Python ≥ 3.10 (используется синтаксис `list[Path]`, `Path | None`).
- `seed = 42` зафиксирован у всех моделей (`np.random.seed`, `random_state`,
`random_seed`).
- Зависимости из протестированного локального окружения:
  ```
  numpy==1.26.4
  pandas==2.2.2
  scipy==1.13.1
  scikit-learn==1.5.2
  catboost==1.2.8
  ```
  В Colab версии могут отличаться — public score возможен ±0.1 от приведённого.
  CatBoost в ноутбуке ставится явно: `pip install catboost==1.2.8`.
- Полный прогон Run All в Colab Free CPU — ~20–30 минут (основное время —
CatBoost на sqrt-ветке).

## Как запустить

В Colab:

1. Открыть `chemai_solution.ipynb`.
2. Положить рядом файлы соревнования: `train.csv`, `test.csv`,
  `sample_submission.csv`.
3. Файл `external_data/chembl_cytotoxicity_cc50_strict_2000_aggregated.csv`
  уже в репозитории — отдельно подкладывать не нужно.
4. `Runtime → Restart session and run all`.

После прогона в `submissions/` появляются:

```
submission_raw_ic2200_si_sqrt_log_w102.csv    # baseline без external
submission_raw_best_cc50ext_b15.csv           # финал для сдачи
submission_raw_best_cc50ext_b22.csv           # альтернативный β = 0.22
```

## О структуре кода

Логика вынесена в функции (`fit_model_runs`, `ensemble_predictions`,
`build_external_cc50_prediction`, ChEMBL-препроцессинг). На раннем этапе работа
велась как обычный Python-пакет с тестами — оттуда в ноутбук перешли type hints
и разбиение по функциям. Для сдачи всё сведено в один `.ipynb`, чтобы прогон
был в одно «Run All».
